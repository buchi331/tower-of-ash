# 『創世のヒント』(仮) ── 設計書 (spec)

- 日付: 2026-06-23
- 種別: 新規ゲーム企画（Unity / 2D / シングルプレイ）
- ステータス: ブレインストーミング合意済み → 実装計画(plan)待ち
- 関連: 各時代リサーチ [docs/god-game/era-research.md](../../god-game/era-research.md)

---

## 1. 一言コンセプト

> あなたは名もなき神。猿人に小さな"ヒント"を授けながら人類を現代文明へ導く。だが授け方を誤れば、世界は思わぬ方向へ歪んでいく。

「流行りの核（文明発展・神視点）＋ 自分のスパイス（神の選択で歴史が良くも悪くも歪む）」という方針で作る。

## 2. 目的・狙い

- **最優先目標: 1本を最後まで完成させる**（Unityの完走経験を得る）。
- 制作者(Naoto)はゲーム開発初心者。物理・通信・3D数学を避け、**データ・UI・選択**で勝負できる構造を採用。
- 制作者の価値観「安易な量産AIコンテンツを嫌う／深み・ゲームフィール重視」に沿い、**史実ベースの本物の重みを持つ選択**で歯ごたえを出す。

## 3. ジャンル/プラットフォーム前提

- 2D・シングルプレイ（マルチプレイヤーは不採用＝1本目の沼を回避）。
- リアルタイム処理なし。**ターン制（1時代＝1ステージ）**。
- エンジン: Unity（2D）。言語: C#。
- 配布形態は将来検討（PC/ブラウザ/モバイル）。MVPはPCで動けばよい。

## 4. コアループ（1時代＝1ステージ）

1. **時代の幕開け**: 中央の「見下ろしの街並みビュー」がその時代の人類の姿を表示。短いナレーション。
2. **岐路（事件）**: 人類が壁にぶつかる出来事が起きる（寒さ・出会い・余剰…）。
3. **授けもの（プレイヤーの選択）**: 2〜3個の"ヒント"から1つ授ける。各ヒントに方向性がある（例: [火] / [毛皮] / [見守る]）。
4. **結果**: 5本のメーターが上下し、街並みビューが変化。良くも、予想外の歪みにも転ぶ。短い結末ナレーション。
5. **次の時代へ**: 蓄積した状態が次の事件・選択肢・展開を変える。
6. **最終時代**で、積み上がった5軸のバランスに応じて**エンディング分岐**。

## 5. メーター（世界の状態＝5本の軸）

`技術` ・ `自然` ・ `平等` ・ `平和` ・ `精神/文化`

- 各メーターは数値（例: 0〜100、初期値は中庸）。授けものの結果で増減する。
- **極端な偏りが"歪み"を生む**（例: 技術↑↑＋自然↓↓ → 公害ディストピア方向）。
- メーターは常時画面上部に表示。**世界の風景（街並みビュー）と連動**して見た目が変わる:
  - 自然↓ → 煙・枯れ・濁りのオーバーレイ
  - 技術↑ → 建物が高度化・増加
  - 平和↓ → 城壁・武具・荒廃の要素
  - 平等↓ → 豪邸とスラムの併存
  - 精神↑ → 神殿・モニュメント・祭礼の要素

## 6. 授けもの・岐路・分岐の仕組み

- 各ステージ＝1つの「岐路イベント」。イベントは**2〜3個の選択肢（授けもの）**を持つ。
- 各選択肢は「結果テキスト＋メーター変化（5軸の増減）＋必要なら状態フラグ」を持つ。
- **分岐**は2層で表現する（軽量に保つ）:
  1. **メーター駆動**: 累積した5軸の値が、後続イベントの分岐や最終エンディングを決める。
  2. **フラグ駆動（任意）**: 特定の選択が後の時代の選択肢/イベント文面を変える（例: 早期に「火を独占」→ 後の宗教ステージで"神権政治"の選択肢が出る）。
- 「**IF分岐（世界の歪み）**」: 各転換点に用意した極端な帰結（リサーチ参照）を、メーター極端化やフラグで発火させ、専用の結果演出を出す。

## 7. ステージ構成（全4幕・推奨18ステージ）

詳細な岐路・選択肢・結果・IF分岐は [era-research.md](../../god-game/era-research.md) に全41転換点を保存済み。そこから歯ごたえ・意外性の高いものを選んだ背骨が以下。★はMVP（最小完成版）に含める6ステージ。

**第1幕 人類の黎明**
- ★直立二足歩行（最初の重い選択）/ ★火の手なずけ / 言葉と象徴 / 犬の家畜化

**第2幕 文明の誕生**
- ★農業革命（"善が呪いになる"主題の提示）/ 都市（ウルク型 vs インダス型）/ ★文字（知識＝支配）/ 組織化された宗教 / 青銅器→鉄器

**第3幕 古典〜中世**
- 枢軸時代の思想 / ゼロの発明 / 火薬（最も純粋な諸刃）/ 黒死病（破滅が解放を生む逆説）

**第4幕 近代〜現代**
- 大航海・コロンブス交換 / ★産業革命（豊かさ vs 公害）/ 近代医学 / ★核分裂（自滅に最も近い）/ ★AI・遺伝子編集（最後の神の決断＝全エンドの分岐点）

## 8. エンディング（最終的な5軸のバランスで分岐）

1. 理想文明（均衡のユートピア）── 5軸が高く均衡。最難関。
2. 飼われた楽園（AI委任）── 技術・平和極大、精神低下。ほろ苦い。
3. 灰の自滅 ── 技術だけ高く他が崩壊。※既存プロジェクト『灰の塔』と世界観が響く。
4. 沸騰する地球（環境崩壊）── 繁栄の果ての窒息。
5. 分裂した種（監視・新カースト）── 技術最高、平等最底。
6. 永遠の静止 ── 封印しすぎた停滞。自然・平和高、技術低。

MVPでは最低2種（例: 「理想文明」寄りと「灰の自滅」寄り）が到達可能であればよい。

## 9. UI / 画面構成（"見た目だけ都市シム"）

単一シーン・パネル切り替えで構成。

- **上部バー**: 現在の幕/時代/年代 ＋ 5本のメーター ＋「時代を進める」。
- **中央**: 見下ろしの街並みビュー（その時代の人類）。**時代と選択で姿が変化**（石器→集落→都市→大都市、メーターに応じたオーバーレイ）。プレイヤーは基本"眺める"。
- **下部**: 岐路イベントの状況テキスト ＋ 授けものの選択肢（2〜3ボタン、各アイコン付き）。選択後は結果テキスト＋メーター変化アニメ。
- 「本格的な都市経営（建物配置・経済シミュレーション）」は**不採用**（核が薄まり、完成が遠のくため）。将来の夢の2本目に回す。

## 10. アート方針（Carto風）

- 方向性: **Carto風**（参考: ゲーム『Carto』）── 温かみのある手描き調イラストの見下ろし世界。柔らかな限定パレット、紙/地図のような質感、ディテールを絞った"クリーンで居心地のよい"様式。「神が"世界という一枚の地図"を見下ろす」テーマと相性が良い。
- 制作者の価値観（量産AIの対極＝"設計された手触り"）に合致。AIによる無造作な量産は不採用。
- 写実は狙わず**様式化（簡潔で統一感のある描き方）**に徹し、初心者でも描き切れる範囲に収める。
- 時代ごとの街並みは「**レイヤー差し替え＋メーター連動オーバーレイ**」（煙・城壁・神殿などの要素を足し引き）で表現し、描く絵の枚数を抑える。
- MVPはCarto風に寄せた簡素な統一スタイルで作る（プロト＝完成品の見た目を兼ねる方針を維持）。本制作の作画は自作または依頼で磨く。

## 11. 技術スタック / 初心者向けの道筋

- **Unity 2D**（最新LTS推奨）、**C#**。
- **データ駆動**: 時代/イベント/選択肢/結果を **ScriptableObject** で定義 → コードを書かずに内容を増やせる（content-heavyかつ初心者向き）。
- **状態管理**: `GameState`（5メーターの数値＋通過フラグ＋履歴）をひとつ持つシンプルな構造。
- **UI**: uGUI（Canvas）を基本とする（チュートリアル資料が最も豊富なため）。
- **シーン**: 単一シーン＋パネルの表示切替。シーン遷移は最小限。
- **セーブ**: 解放したエンディング等は JSON か PlayerPrefs（MVPでは省略可）。
- **不採用**: ネットワーク、物理、経路探索、リアルタイムシミュレーション、3D。
- 学習は「公式の2D入門 → ScriptableObjectでデータ駆動 → uGUIでイベント/選択UI」の順に、本企画を題材に進める。

## 12. データ構造（方針スケッチ）

```
EraSO (時代)
  - id, 表示名, 年代ラベル, 幕
  - 街並みビューの基本ビジュアル参照
  - このEraで発火しうる EventSO のリスト

EventSO (岐路)
  - id, 状況テキスト
  - 出現条件（必要フラグ / メーター条件 / 既定）
  - ChoiceSO のリスト（2〜3）

ChoiceSO (授けもの)
  - 表示名, アイコン, 一言
  - 結果テキスト
  - メーター変化（技術/自然/平等/平和/精神 の増減）
  - 付与フラグ（任意）
  - IF演出トリガー（任意：メーター極端化等）

GameState (実行時)
  - 5メーターの現在値
  - 通過フラグの集合
  - 選択履歴（振り返り/エンディング判定用）

EndingSO (結末)
  - id, 表示名, 本文
  - 判定条件（最終5軸のバランス）
```

## 13. 完成の定義（MVPのDefinition of Done）

- ★6ステージ（直立二足歩行→農業→文字→産業革命→核分裂→AI）が**頭から尻尾まで通しで遊べる**。
- 5メーターが選択で増減し、画面上に見える。
- 街並みビューが6時代で変化する（簡素でよい）。
- 最低2種のエンディングに到達できる。
- 選択に「目に見える結果（テキスト＋メーター＋見た目）」が伴う。

## 14. 段階計画（概略）

1. **MVP**: 上記6ステージ＋5メーター＋街並み変化＋2エンディングで「ループ全体」を完走。
2. **肉付け**: 残り12ステージ、フラグ駆動の分岐、IF演出、6エンディングへ拡張。
3. **磨き**: アート差し替え、音、演出、テキスト推敲、バランス調整。

## 15. リスクと対策

- **コンテンツ量が多い** → ScriptableObjectでデータ駆動化し、MVPは6ステージに限定。
- **アートで手が止まる** → MVPは意図的ミニマル様式で凍結。絵の差し替えは後工程。
- **分岐の複雑化で破綻** → MVPはメーター駆動のみ。フラグ駆動は肉付け段階で慎重に追加。
- **完璧主義による未完** → 「完成の定義」を満たしたら一度"完成"とみなし、その後に磨く。
