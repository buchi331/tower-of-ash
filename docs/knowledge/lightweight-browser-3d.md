- **状態**: 調査メモ（実物のライブ計測は未実施 / 公開情報＋OSS再現版から構成）
- **元ネタ**: `messenger.abeto.co`（studio abeto / Awwwards Site of the Day）

---

## TL;DR

「ダウンロード不要のURLひとつで、ゲーム級の3D体験がブラウザで動く」を成立させる鍵は、
**“軽さ”を絵作り・圧縮・読み込みの全レイヤーで一貫して設計すること**。
見た目の良さはデータ量ではなく **アートディレクション＋シェーダー＋ポストFX（＝ほぼ無料のバイト数）** で出すので、「軽い」と「キレイ」がトレードオフでなくなる。

---

## 1. 何を達成しているか

`messenger.abeto.co` は、小さな球体の惑星で郵便配達をする**マルチプレイ・ブラウザゲーム**。

- **初期ロード 5.7MB / 最大 17.5MB**、数秒で起動、インストール不要
- PC/スマホ同一URL、スマホでも滑らか
- Awwwards 受賞、SNSで「これがただのウェブサイト？」とバズった

比較として、Unity の WebGL ビルドは普通 30〜100MB超。**だから彼らは Unity/Godot を使わず Three.js でゼロから自作している**。

## 2. 技術の6本柱

| やりたいこと | 使った技術 | 効果 |
|---|---|---|
| 即起動・無インストール | Draco圧縮（メッシュ~1/10）＋ローポリ＋段階ロード | 5.7MBで「ただのURL」 |
| 独特の美しい絵 | 大量の自前GLSLセルシェーダー＋ポストプロセス | 量産Three.js感の払拭 |
| 曲面の惑星を滑らかに歩く | `three-mesh-bvh`（衝突/レイキャスト高速化） | 毎フレーム衝突でも60fps |
| 球体の重力 | 中心方向への自作up-vector（物理エンジン不使用） | 軽量・自由な惑星表現 |
| 一緒にいる感 | Node.js上のWebSocket（座標＋絵文字のみ同期） | 低帯域・モデレーション容易 |
| アセット制作 | Houdini＋Blenderでモデル、Substanceでテクスチャ | 作り込み |

## 3. 見た目の作り方（ここが核心）

**シェーダー = 画面の各ピクセルを「何色にするか」その場で計算するGPU上の小プログラム。**
普通は「貼った画像をそのまま表示」だが、この作品は光・影・色をすべてプログラムで指示している。
（有志のOSS再現版 [arafays/messenger-copy](https://github.com/arafays/messenger-copy) は**コードの79%がGLSL** ＝ 絵作りがシェーダーに集中している証拠。）

1. **セルシェード**: 現実のなめらかな光を**わざと数段階のベタ塗り＋輪郭線**にする → アニメ/絵本調。
2. **素材ごとの専用シェーダー**: 水・砂・草を個別に手書き（開発者が「ビーチのシェーダーに時間をかけすぎた」と発言）。
3. **ポストプロセス**: 描き終えた画面全体に **ブルーム（光のにじみ）＋カラーグレーディング（色味統一）＋周辺減光** をかけ、世界観を一つにまとめる。

> たとえ: ふつうの3D＝現実を「写真に撮る」（高精細＝重い）／この作品＝絵描きが「一筆ずつ光と色を塗る」（やり方しだいで軽くてキレイ）。

## 4. なぜ「軽い」と「キレイ」が両立するのか

**キレイさ＝細かさ（データ量）ではなく、光と色のセンス＋シェーダーの工夫で出している。**
写実で頑張ると重くなるが、彼らは「絵としての美しさ」で勝負したのでデータは小さいまま見た目だけリッチにできる。
＝ **「軽さ」は制約ではなく、全レイヤーを貫く設計目標**。

## 5. 自分で作るときのチェックリスト（再利用レシピ）

- [ ] Unity/Godotに逃げず、まず **Three.js** で必要な部品だけ作れないか検討する
- [ ] モデルは **Draco** で圧縮、絵柄は**ローポリ＋トゥーン**寄りにしてデータを小さく保つ
- [ ] 見た目の勝負どころは**カスタムGLSL**（トゥーン陰影・輪郭線・素材別の塗り）に投資する
- [ ] 仕上げに**ポストプロセス**（ブルーム/カラグレ/ビネット）で一気に世界観を統一する
- [ ] 地形の歩行・当たり判定は **three-mesh-bvh** で高速化して60fpsを死守する
- [ ] マルチは **WebSocket** で「最小限の状態（座標＋絵文字程度）」だけ同期する
- [ ] 「初期ロードを軽く→残りを裏で**段階ロード**」で体感速度を稼ぐ
