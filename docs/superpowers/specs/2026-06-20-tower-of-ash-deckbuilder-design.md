# 灰の塔（Tower of Ash）— 設計仕様書 v1

- 日付: 2026-06-20
- ジャンル: デッキ構築ローグライク（Slay the Spire ライト）
- 世界観: ダークファンタジー
- 仮タイトル: 灰の塔 / Tower of Ash

---

## 1. ビジョンと狙い

塔を登るほど敵が強くなる一本道のローグライク。プレイヤーは**手作りで設計されたカード**を集めてデッキを組み、ターン制バトルで敵を倒して最上階のボスを目指す。

面白さの源泉は「無限のランダム生成」ではなく、**設計されたカード同士のシナジー・敵の予告を読む駆け引き・1回の冒険のヒリつき・"もう1回"の中毒性**。トレカ好きが「設計されたカードを集めて組む」快感に直球で応える。

### この企画が満たすべき制約（不変）
- **AI生成ゼロ / API料金ゼロ** — カード・敵はすべて人が設計したデータ。
- **完全クライアントサイド / サーバー無し / オフライン動作。**
- **PWA（静的Webアプリ）** — URL一発で誰でも即プレイ、ホーム画面追加可。
- **スマホ縦持ち・タッチ前提。**
- 無料配信。課金沼にしない。

---

## 2. プラットフォームと技術スタック

- ビルド: **Vite + React + TypeScript**（静的ビルド → 任意の静的ホストへデプロイ、URL共有可）
- 保存: **localStorage**（スキーマのバージョン管理付き）
- PWA: manifest + Service Worker（ホーム追加・オフライン）
- アニメ/手応え: CSS トランジション中心（必要に応じ軽量ライブラリ）
- 乱数: **シード可能な自前RNG**（テスト再現性・将来のデイリーシード共有に備える）

理由: コンポーネント分割しやすく、型でバグを防げ、学習資料が最多で初心者が伸ばしやすい。後の「本格化（Godot/Unity/3D）」にも繋がる標準スキル。

---

## 3. 中核：戦闘ループ

ターン制。プレイヤーは HP・エネルギー・山札（draw / hand / discard）を持つ。

1. ターン開始時、手札を **5枚** 引く。エネルギーが **3** に回復。
2. 敵は**次の行動を予告**する（intent: 「攻撃12」「防御」「脆弱付与」等）。→ プレイヤーは先読みできる。
3. プレイヤーはエネルギーを払ってカードをプレイ（攻撃 / 防御=ブロック / 効果）。
4. 「ターン終了」→ 敵が予告通り行動 → 状態異常を処理 → 1 に戻る。

### ルール詳細
- **ブロック**は被ダメージを肩代わりし、**毎ターン開始時にリセット**（Slay the Spire 方式）。
- 手札は山札が尽きたら捨札をシャッフルして補充。
- カードをプレイすると捨札へ（Power=持続カードは場に残る）。

### 状態異常は4種のみ（合意済み）
| 名称 | 効果 | 減衰 |
|---|---|---|
| 脆弱 (Vulnerable) | 受ける攻撃ダメージ +50% | 毎ターン -1 |
| 弱体 (Weak) | 与える攻撃ダメージ -25% | 毎ターン -1 |
| 力 (Strength) | 攻撃カードのダメージ +N（戦闘中持続） | 減衰なし |
| 毒 (Poison) | ターン開始時 N ダメージ、その後 N を -1 | 値が減衰 |

この4種だけで「毒型・力型・防御反撃型」のシナジーが十分に組める。

---

## 4. 冒険（ラン）の構造

- 塔を**一本道で8階**登る（v1はマップ分岐なし）。
  - 例: 1〜3階=通常戦、4階=エリート、5〜6階=通常戦（強化）、7階=エリート、8階=ボス。
- **HPはラン中ずっと持ち越し。**
- 各**非ボス戦の勝利後、報酬として3枚から1枚を選んでデッキに加える**（スキップ可）。加えて少量回復（初期値: エリート撃破時に最大HPの25%回復。要調整）。
- **ボス撃破=クリア。HP0=ラン終了 → 初期デッキでやり直し**（ローグライク）。

## 5. コンテンツ（v1）

数値はすべて**初期値（要バランス調整）**。

### クラス: 呪われた騎士
- 開始HP: 70
- 初期デッキ（10枚）: 斬撃×5、守勢×4、痛打×1

### カードプール（約20枚 / 3アーキタイプ）
攻撃:
- 斬撃 — 1E / 6ダメージ（基本）
- 痛打 — 2E / 8ダメージ + 脆弱2
- 連刃 — 1E / 3ダメージ×2回（力と好相性）
- 重撃 — 2E / 14ダメージ
- 毒刃 — 1E / 5ダメージ + 毒3
- 吸血 — 2E / 8ダメージ + 4回復
- 渾身 — 3E / 22ダメージ（フィニッシャー）
- 捨て身 — 0E / 10ダメージ + 自分に3ダメージ（リスク&リターン）
- 盾打ち — 1E / 現在のブロック値と同じダメージ（防御→攻撃へ転換）

防御/効果:
- 守勢 — 1E / 5ブロック
- 鉄壁 — 2E / 12ブロック
- 集中 — 1E / 力+2

持続（Power）:
- 鍛錬 — 1E / 力+3（戦闘中持続）
- 毒の心得 — 1E / 攻撃カードをプレイするたび毒+1（毒型エンジン）

ドロー/経済:
- 戦機 — 0E / 2枚引く + エネルギー+1（コンボの起点）
- 見極め — 1E / 2枚引く

（上記＋斬撃/守勢/痛打の初期カードで合計約20種。バランス調整で増減）

### 敵（通常5〜6 + エリート + ボス）
各敵は HP と**予告パターン**を持つ。
- 骸骨剣士: 攻撃6〜9 / たまに防御
- 蝙蝠群: 小ダメージ多段 + 弱体付与
- 泥人形: 高HP / 1ターンおきに大ダメージを予告
- 屍喰い: プレイヤーに毒付与
- 影: ブロックを得つつドレイン
- 〔エリート〕呪術師: 脆弱付与 + 中ダメージ
- 〔エリート〕鎧の番人: 高ブロック + 大ダメージ
- 〔ボス〕灰の王: HP約120 / 「大攻撃 → 弱体付与 → ブロック&回復」を周回する多段構成

## 6. カードの見た目（チープにしない工夫）

ダークファンタジーのカード枠を **CSS + アイコン（Tabler等）+ 色分け**で表現。
- 攻撃=赤系 / 防御=青系 / 効果・Power=紫系。
- コスト（左上）・名前・効果テキスト・タイプ色帯。
- イラストが無くても「意図された」見た目になり安っぽくならない。
- 絵は**後で差し替え可能な構造**（カード定義に art 指定フィールドを持たせる）。

---

## 7. 手応え（"つまらない"対策・第一級要件）

実装の後回しにせず最初から作り込む:
- 敵の**予告**を明確に表示（先読みできる＝駆け引きが生まれる）。
- **ダメージ数字のポップ**、カードプレイ時の動き、被弾のシェイク、**ブロック割れ**の表現。
- カード選択が**意味を持つ**（明確な強み・弱み・シナジー）。
- 1ターンの操作がキビキビ反応する（入力遅延ゼロ）。

## 8. アーキテクチャ（独立した部品 / well-defined interface）

| モジュール | 責務 | 依存 |
|---|---|---|
| `model`（型定義） | Card / Enemy / Intent / Status / CombatState / RunState / PlayerState のデータ型 | なし |
| `content`（データ） | カード定義・敵定義・階層構成。ロジックと分離（バランス調整＝データ編集） | model |
| `rng` | シード可能な乱数（シャッフル・報酬・敵変動） | なし |
| `combat`（戦闘エンジン） | 純粋関数群: カード適用 / 敵ターン / 状態異常処理 / 勝敗判定。決定論的でテスト可能 | model, rng |
| `run`（ラン制御） | 階層の進行・報酬・ラン状態の保持と更新 | model, combat, content, persistence |
| `persistence` | localStorage 読み書き（バージョン管理・破損時は安全に初期化） | model |
| `ui`（React） | 戦闘画面 / カード / 報酬画面 / 塔の進行 / デッキ確認 / 勝敗画面 | model, run |

設計原則: `combat` と `rng` は**副作用なしの純粋関数**にし、UIから切り離す。これにより単体テストで品質を担保し、UIを差し替えても中身が壊れない。

### データモデル（概略）
- `CardDef { id, name, type: 'attack'|'skill'|'power', cost, effects[], art }`
- `EnemyDef { id, name, maxHp, intentPattern[] }`
- `Intent { kind: 'attack'|'defend'|'buff'|'debuff', value, status? }`
- `StatusEffects { vulnerable, weak, strength, poison }`（対象ごと）
- `CombatState { player, enemy, drawPile, hand, discardPile, energy, turn }`
- `RunState { floor, playerHp, maxHp, deck[], seed, schemaVersion }`

---

## 9. 保存（persistence）

- `localStorage` に「進行中のラン」「設定」を保存（`schemaVersion` 付き）。
- 破損・バージョン不一致時は安全に初期化（クラッシュさせない）。

---

## 10. テスト方針

- **戦闘エンジン（純粋関数）を単体テストで厚く守る**: ダメージ計算・ブロック・各状態異常・各カード効果・勝敗判定・乱数のシード再現。
- UI と全体バランスはプレイテストで確認（手応え・難度曲線）。

## 12. 成功基準

- スマホのブラウザで、ワード不要・タップ操作だけで1ランを最後まで遊べる。
- 「敵の予告を読んでカードを選ぶ」駆け引きが成立し、勝敗が**運だけでなく選択**で決まる。
- 1ラン10〜20分で完結し、負けても「もう1回」と思える。
- 完全オフライン・無料で動作し、URL共有で他人もすぐ遊べる。
